# 💡 ビジネス・インサイト 生成ルール (Prompt Template)

AIエージェントが `daily_report.md` の「ビジネス・インサイト（本日の急騰銘柄と注目ニッチ市場）」セクションを生成する際は、以下のルールを厳格に遵守すること。（**ハルシネーション防止のための絶対ルール**）

## 1. 必須ワークフロー（厳守）
AIは**絶対に**自身の知識やWeb検索のみで急騰銘柄を決定してはならない。以下のステップを必ず踏むこと。

1. **データ取得**: 必ずターミナルで `docker-compose exec trading-bot python fetch_market_movers.py` を実行し、客観的な「本日の急騰トップ銘柄リスト」を取得する。
2. **調査範囲の限定**: スクリプトが返した具体的なティッカーシンボル（例: `PENG`, `WULF`）に限定して、その銘柄がなぜ急騰したのかのニュース（カタリスト）をWeb検索ツールで調査する。
3. **合成**: スクリプトの正確な数値（上昇率・セクター）と、Web検索で裏付けが取れたカタリストだけを用いてレポートを生成する。架空の数値を捏造した場合は即時エラーとみなす。

## 2. 目的とスタンス
- **文脈の補完:** 単なるニュースの直訳ではなく、その急騰事象の裏にある「なぜ上がったのか（カタリスト）」や「その企業が属するニッチ市場の仕組み」を、日本のビジネスパーソンが読んで**新たなビジネストレンドの学び**になるように合成・解説すること。
- **トーン＆マナー:** 客観的（フラット）でプロフェッショナルなトーンを維持する。AI特有の過剰な修飾語（ヨイショ、過大評価）や無駄な前置きは徹底的に排除すること。

## 3. フォーマット制限（厳守）
- Markdownの「テーブル（表）」は使用しない。
- セクションタイトルや指定された項目名以外で、過剰に太字（Bold）を使用しない。
- リスト形式でスッキリとまとめる。

## 4. 出力テンプレート
必ず以下の構造と記法に沿って出力すること。

```markdown
### 💡 ビジネス・インサイト（本日の急騰銘柄と注目ニッチ市場）
> Pythonスクリプト(`fetch_market_movers.py`)を用いて米国市場全体から抽出した本日の急騰トップ銘柄の実データに対し、AIが文脈を補完して新たなビジネストレンドを解説します。

**📌 今日の支配的テーマ:** [ここに、上位銘柄に共通するセクターの傾向や、その日資金が流入したテーマ（例：データセンター関連、量子コンピュータ等）を1〜2文で総括する]

1. **[ティッカーシンボル] (+[上昇率]%)** - セクター: [セクター名] / 産業: [産業名]
   - **急騰の理由(直近ニュース):** [なぜ急騰したのか。事実ベースのカタリスト（例：好決算、大型受注、インサイダー買い等）と、投資家の期待値をフラットに解説]
   - **事業モデル:** [その企業がどのようにマネタイズしているか。どのようなニッチな技術・サービスを持っているか]

2. **[ティッカーシンボル] (+[上昇率]%)** - セクター: [セクター名] / 産業: [産業名]
   - **急騰の理由(直近ニュース):** [...]
   - **事業モデル:** [...]
```
（※以降、上位3〜5銘柄程度を抽出して記載）
